📌 概述

特征工程模块将SDK的原始数据（导入导出序列、操作码序列、TLSH哈希）转换为可用于聚类的数值特征矩阵。整个过程包括数据加载、特征提取、特征融合、标准化和可选的PCA降维。

💡 TF-IDF 原理

TF (Term Frequency): 词在文档中的频率，衡量词在单个文档中的重要性

IDF (Inverse Document Frequency): 词在整个语料库中的稀有程度，衡量词的区分能力

TF-IDF = TF × IDF: 综合衡量词的重要性

🔐 TLSH哈希处理流程
TLSH解析步骤
graph TD
    Hash[TLSH哈希字符串] --> CheckFormat{格式检查}
    
    CheckFormat -->|不是TLSH| Skip[跳过]
    CheckFormat -->|是TLSH| RemovePrefix[去掉'T1'前缀]
    
    RemovePrefix --> HexStr[十六进制字符串]
    HexStr --> ToBytes[转为字节]
    
    ToBytes --> ByteLoop{遍历字节}
    ByteLoop --> GetByte[获取字节值]
    GetByte --> High4[(高4位 >> 4 & 0xF)]
    GetByte --> Low4[(低4位 & 0xF)]
    
    High4 --> Vector70[70维向量]
    Low4 --> Vector70
    
    Vector70 --> CheckCount{有多个哈希?}
    CheckCount -->|是| Average[求平均值]
    CheckCount -->|否| Keep[保持原值]
    
    Average --> Output[TLSH特征向量]
    Keep --> Output

🔍 TLSH哈希解析示例

输入: "T1A521981D0779E0E..." 1. 去掉"T1"前缀: "A521981D0779E0E..." 2. 转为字节: [0xA5, 0x21, 0x98, 0x1D, 0x07, 0x79, ...] 3. 拆分为4位: 字节0xA5 → [10, 5] → [0xA, 0x5] 字节0x21 → [2, 1] → [0x2, 0x1] 字节0x98 → [9, 8] → [0x9, 0x8] ... 4. 输出70维向量: [10, 5, 2, 1, 9, 8, ...]

📏 标准化与PCA流程

⚙️ 标准化方式对比

方法 公式 输出范围 特点
StandardScaler (x - μ) / σ ~N(0,1) 适合正态分布数据，去除量纲影响
MinMaxScaler (x - min) / (max - min) [0,1] 保留原始分布，压缩异常值影响

  3. 查看处理结果: data/processed/processed_features.json
