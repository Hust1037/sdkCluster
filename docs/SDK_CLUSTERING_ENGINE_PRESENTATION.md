# SDK聚类引擎技术方案
## HarmonyOS应用SDK信誉检测解决方案

## 幻灯片1：项目背景与目标

### 业务背景
- 鸿蒙应用生态快速发展，SDK数量快速增长
- 未知SDK存在潜在安全风险
- 需要建立SDK信誉检测体系

### 核心目标
- 对未知SDK进行智能聚类
- 发现潜在的SDK家族和变种
- 建立SDK分类体系
- 为后续规则挖掘提供基础

### 技术挑战
- 多源异构数据处理（文本+数值+序列）
- 高维特征融合（4000+维度）
- 无监督学习场景（缺乏标签数据）
- 大规模样本处理（7000+样本）

## 幻灯片3：特征处理详解

### 六维特征融合架构

```
输入: SDK JSON数据
  ├─ coordinateName: "@abner/log"
  ├─ version: "1.0.3"
  └─ codeTlshHashes: [序列1, 序列2, ...]
          │
          ▼
    智能分类器
  ├─ TLSH Hash (70字符)
  ├─ Import/Export (含':')
  ├─ Opcode (指令关键词)
  ├─ Business Semantic (业务常量)
  ├─ Coordinate Name (SDK名称)
  └─ Numeric Sequence (数字序列)
          │
          ▼
    特征提取
  ├─ Doc2Vec (深度学习) - Opcode
  ├─ TF-IDF (文本挖掘) - Import/Export/Semantic/Coordinate
  ├─ 数值转换 - TLSH
  └─ 统计分析 - Numeric
          │
          ▼
    特征融合 (加权)
  ├─ Coordinate: 15%
  ├─ Semantic: 15%
  ├─ Import/Export: 15%
  ├─ Opcode: 35% (最高权重)
  ├─ TLSH: 10%
  └─ Numeric: 10%
          │
          ▼
    归一化处理
  ├─ MinMaxScaler (文本特征)
  └─ StandardScaler (数值特征)
          │
          ▼
输出: 6416维特征矩阵
```

### Doc2Vec深度学习优势

| 传统TF-IDF | Doc2Vec |
|----------|----------|
| 词袋模型 | 语义嵌入 |
| 无上下文 | 保留指令序列 |
| 固定维度 | 动态学习 |
| 线性组合 | 非线性语义 |

**技术参数**:
- 向量维度: 128
- 上下文窗口: 5
- 训练迭代: 20 epochs
- 并行训练: 4 workers

### 特征权重分配策略

```python
# 业务驱动权重分配
config = {
    'opcode_weight': 0.35,       # 指令逻辑 - 最重要
    'import_weight': 0.15,       # 依赖关系
    'semantic_weight': 0.15,      # 业务功能
    'coordinate_weight': 0.10,   # 命名模式
    'tlsh_weight': 0.10,          # 代码相似性
    'numeric_weight': 0.10        # 补充统计
}
```

**权重设计理由**:
- Opcode权重最高: 最能反映SDK核心行为
- 语义特征重要: 捕获业务功能差异
- 名称权重适中: 避免过度依赖命名

## 幻灯片4：聚类算法对比

### 算法特性对比

| 算法 | 聚类数量 | 噪声处理 | 参数数量 | 计算复杂度 | 业务适配性 |
|------|----------|----------|----------|----------|----------|
| **HDBSCAN** | 自动 | 强 | 2 | O(n²) | ⭐⭐⭐⭐⭐⭐ |
| **DBSCAN** | 自动 | 强 | 2 | O(n log n) | ⭐⭐⭐⭐ |
| **K-Means** | 需指定 | 弱 | 3 | O(n × k × i) | ⭐⭐ |
| **Agglomerative** | 需指定 | 弱 | 3 | O(n³) | ⭐⭐ |

### HDBSCAN推荐理由

**1. 自动聚类数量**
- 无需预先知道SDK类别数量
- 适应未知SDK的探索性分析
- 降低业务人员使用门槛

**2. 强大噪声识别**
- 自动标记异常SDK
- 为信誉检测提供预警机制
- 支持后续人工审核

**3. 层次结构发现**
- 识别SDK的父子关系
- 发现功能变种和版本演进
- 建立SDK分类体系

**4. 复杂分布适应**
- 不受球形假设限制
- 适应SDK的多样分布
- 发现自然聚类边界

**5. 参数鲁棒性**
- 主要参数少(min_cluster_size)
- 对参数变化不敏感
- 结果稳定性高

### 算法参数对比

| 参数 | HDBSCAN | DBSCAN | K-Means | Agglomerative |
|------|----------|---------|----------|---------------|
| **聚类数量** | 自动 | 自动 | 指定k | 指定k |
| **核心参数** | min_cluster_size | eps, min_samples | n_clusters | n_clusters |
| **距离度量** | 可配置 | 可配置 | 欧式距离 | 可配置 |
| **层次结构** | 有 | 无 | 无 | 有 |

## 幻灯片5：评估指标体系

### 业务导向评估框架

```
┌─────────────────────────────────────────────────────────────┐
│              Ground Truth构建                         │
│  按coordinateName分组 → SDK家族识别            │
│  同一SDK多版本 → 基准标签                  │
└──────────────────┬──────────────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────────────────────────┐
│              聚类结果验证                         │
│  ┌──────────────────────────────────────────────┐      │
│  │ 同质性检查: 同一SDK是否分到同一类      │      │
│  ├─ 提取同一家族的样本                    │      │
│  ├─ 检查聚类标签是否一致                    │      │
│  └─ 计算Homogeneity Rate                   │      │
│  └──────────────────────────────────────────────┘      │
│  ┌──────────────────────────────────────────────┐      │
│  │ 噪声分析: 识别异常SDK                 │      │
│  ├─ 统计标签为-1的样本                    │      │
│  └─ 计算Noise Rate                         │      │
│  └──────────────────────────────────────────────┘      │
│  ┌──────────────────────────────────────────────┐      │
│  │ 聚类质量: 轮廓系数和分离度              │      │
│  ├─ 计算Silhouette Score                   │      │
│  ├─ 计算Davies-Bouldin Score               │      │
│  └─ 计算Calinski-Harabasz Score            │      │
│  └──────────────────────────────────────────────┘      │
└─────────────────────────────────────────────────────────────┘
```

### 评估指标说明

| 指标 | 计算方式 | 取值范围 | 业务意义 | 目标值 |
|------|----------|----------|----------|--------|
| **Cluster Count** | len(set(labels)) | [1, n] | 发现的SDK类别数 | 覆盖业务范围 |
| **Noise Rate** | count(-1)/n | [0, 1] | 异常SDK比例 | < 10% |
| **Homogeneity** | 匹配率 | [0, 1] | 同一SDK聚合正确性 | > 80% |
| **Silhouette** | sklearn | [-1, 1] | 聚类分离度 | > 0.5 |
| **DB Index** | sklearn | [0, ∞) | 聚类紧凑度 | < 1.0 |
| **CH Index** | sklearn | [0, ∞) | 聚类分离度 | 越大越好 |

### 评估流程

```python
# 1. 构建Ground Truth
ground_truth = build_sdk_families(sample_info)

# 2. 运行聚类算法
labels = hdbscan.fit_predict(features)

# 3. 计算业务指标
metrics = {
    'cluster_count': count_clusters(labels),
    'noise_rate': calculate_noise(labels),
    'homogeneity': check_homogeneity(labels, ground_truth),
    'silhouette': silhouette_score(features, labels)
}

# 4. 导出基准家族
export_baseline_families(ground_truth, 'output/baseline.json')
```
